## 🚀 微信云托管配置步骤

### 第一步：进入云托管控制台

1. **访问云托管**
   - 打开浏览器
   - 访问：https://cloud.weixin.qq.com/
   - 登录你的微信账号

2. **进入服务列表**
   - 点击左侧 **"服务管理"** → **"服务列表"**

---

### 第二步：创建新服务（或配置现有服务）

#### 选项 A：创建新服务（推荐）

1. **点击"新建服务"**
   - 在服务列表页面
   - 点击右上角 **"新建服务"** 或 **"+"** 按钮

2. **填写基本信息**
   | 配置项 | 填写内容 |
   |--------|----------|
   | **服务名称** | `wechat-bot` |
   | **服务类型** | Web服务 |
   | **实例类型** | 免费版（或基础版） |

3. **选择部署方式**
   - 选择 **"代码仓库"**
   - 点击 **"下一步"**

4. **配置代码仓库**
   | 配置项 | 填写内容 |
   |--------|----------|
   | **代码来源** | GitHub |
   | **仓库地址** | `https://github.com/你的用户名/wechat-bot` |
   | **分支** | `main` |

5. **授权 GitHub**
   - 如果第一次使用，点击 **"授权"**
   - 使用 GitHub 账号登录
   - 授权微信云托管访问你的仓库
   - 选择你的 `wechat-bot` 仓库

6. **配置构建信息**
   | 配置项 | 填写内容 |
   |--------|----------|
   | **构建命令** | 留空（自动检测） |
   | **启动命令** | 留空（使用 Procfile） |
   | **端口** | `80` |

7. **点击"下一步"**

---

#### 选项 B：配置现有服务

如果你不想创建新服务，可以配置现有的 `flask-enou` 服务：

1. **进入服务详情**
   - 在服务列表中找到 `flask-enou`
   - 点击进入

2. **进入"服务设置"**
   - 点击顶部的 **"服务设置"** 标签

3. **配置代码仓库**
   - 找到 **"代码仓库"** 或 **"源码管理"**
   - 点击 **"更换"** 或 **"编辑"**
   - 按照选项 A 的步骤 4-7 配置

---

### 第三步：配置环境变量（最重要！⚠️）

**这一步必须正确配置，否则服务无法调用 Coze AI！**

#### 3.1 进入环境变量配置

在服务配置页面：
1. 找到 **"环境变量"** 或 **"配置"** 部分
2. 点击 **"添加环境变量"**

#### 3.2 添加必需的环境变量

逐个添加以下变量：

**变量 1：工作目录**
```
名称: COZE_WORKSPACE_PATH
值: /workspace/projects
```

**变量 2：Coze API 密钥** ⚠️ 最重要
```
名称: COZE_WORKLOAD_IDENTITY_API_KEY
值: 你的Coze API密钥
```

**如何获取 Coze API 密钥：**
1. 登录 Coze 平台
2. 进入你的项目
3. 点击 **"设置"** 或 **"API Key"**
4. 复制 **API Key**
   - 格式：`pat_xxxxxxxxxxxxxxxxxxxxxxxx`

**变量 3：Coze Base URL**
```
名称: COZE_INTEGRATION_MODEL_BASE_URL
值: 你的Coze Base URL
```

**可能的值：**
- `https://api.coze.com/v1`（国际版）
- `https://api.coze.cn/v1`（国内版）

**变量 4：超时时间**
```
名称: UV_HTTP_TIMEOUT
值: 300
```

#### 3.3 验证环境变量

添加完成后，确认：
- ✅ 所有 4 个变量都已添加
- ✅ 变量名拼写正确（区分大小写）
- ✅ API 密钥正确复制

---

### 第四步：部署服务

1. **检查所有配置**
   - 代码仓库：✅
   - 环境变量：✅
   - 端口：✅

2. **点击"部署"**
   - 在服务配置页面
   - 点击 **"部署"** 或 **"发布"** 按钮

3. **等待部署**
   - 通常需要 3-5 分钟
   - 查看部署进度
   - 等待 "部署成功" 提示

4. **查看部署日志**
   - 如果失败，查看日志
   - 检查错误信息
   - 修复问题后重新部署

---

### 第五步：获取公网 URL

部署成功后：

1. **进入服务详情**
   - 点击 **"服务管理"** → **"服务列表"**
   - 找到你的服务（`wechat-bot`）
   - 点击进入

2. **查看访问配置**
   - 在服务详情页面
   - 找到 **"访问配置"** 或 **"公网访问"**
   - 复制 **公网访问地址**

**URL 格式：**
```
https://你的服务名.cloudrun.tencentcloudapi.com
```

**示例：**
```
https://wechat-bot.cloudrun.tencentcloudapi.com
```

---

### 第六步：配置微信公众号

#### 6.1 进入公众号后台

1. **登录微信公众号**
   - 访问：https://mp.weixin.qq.com/
   - 使用你的微信扫码登录

2. **进入服务器配置**
   - 点击左侧 **"开发"** 菜单
   - 点击 **"基本配置"**
   - 点击 **"服务器配置"** 部分

#### 6.2 填写配置信息

**重要：** URL 必须包含 `/stream_run` 路径！

| 配置项 | 填写内容 |
|--------|----------|
| **URL** | `你的云托管URL/stream_run` |
| **Token** | `xingtuai2026` |
| **EncodingAESKey** | 点击"随机生成" |
| **消息加密** | 选择 **明文模式** |
| **数据格式** | XML |

**示例：**
```
URL: https://wechat-bot.cloudrun.tencentcloudapi.com/stream_run
Token: xingtuai2026
EncodingAESKey: 点击"随机生成"
消息加密: 明文模式
数据格式: XML
```

#### 6.3 提交配置

1. **填写完所有配置后**
2. **点击"确定"按钮**
3. **等待验证**

**如果成功：**
```
服务器配置已生效
```

**如果失败：**
- 检查 URL 是否正确（包含 `/stream_run`）
- 检查 Token 是否为 `xingtuai2026`
- 检查云托管服务是否正在运行
- 查看云托管日志

---

### 第七步：测试消息收发

1. **用另一个微信关注你的公众号**
   - 扫描公众号二维码
   - 或搜索公众号名称关注

2. **发送测试消息**
   - 发送："你好"
   - 发送："介绍一下你自己"
   - 发送："帮我搜索一些信息"

3. **观察回复**
   - 如果收到 AI 回复 → ✅ 成功！
   - 如果没有回复 → 查看云托管日志

## 🔑 关键配置说明

### 1. Coze API 密钥（最重要！）

**为什么需要：**
- 服务需要调用 Coze 智能体
- 没有这个密钥，服务会报错
- AI 功能无法使用

**如何获取：**
1. 登录 Coze 平台
2. 进入项目设置
3. 找到 API Key
4. 复制密钥

**格式：**
```
pat_xxxxxxxxxxxxxxxxxxxxxxxx
```

### 2. URL 路径（容易出错！）

**必须包含：**
```
/stream_run
```

**正确：**
```
https://wechat-bot.cloudrun.tencentcloudapi.com/stream_run ✅
```

**错误：**
```
https://wechat-bot.cloudrun.tencentcloudapi.com ❌
```

### 3. Token（必须一致）

**两个地方必须相同：**
- 服务器代码：`xingtuai2026`
- 微信配置：`xingtuai2026`

## ❌ 常见错误及解决方案

### 错误1：部署失败

**原因：**
- 代码仓库无法访问
- 依赖安装失败
- 环境变量错误

**解决：**
1. 检查 GitHub 仓库是否公开
2. 检查 `requirements.txt` 是否正确
3. 检查环境变量是否正确

### 错误2：微信配置失败（HTTP返回非200）

**原因：**
- URL 不正确
- Token 不匹配
- 服务未运行

**解决：**
1. 检查 URL 是否包含 `/stream_run`
2. 检查 Token 是否为 `xingtuai2026`
3. 检查云托管服务是否正在运行
4. 查看云托管日志

### 错误3：服务运行但无法回复

**原因：**
- Coze API 密钥错误
- 环境变量未配置
- 服务调用失败

**解决：**
1. 检查 `COZE_WORKLOAD_IDENTITY_API_KEY` 是否正确
2. 检查 `COZE_INTEGRATION_MODEL_BASE_URL` 是否正确
3. 查看服务日志，查看具体错误
