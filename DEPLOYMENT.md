# 微信云托管部署指南

## 📋 部署前检查清单

### ✅ 已修复的问题

1. **zstandard 下载超时**
   - 降级版本：`0.25.0` → `0.23.0`
   - 原因：0.25.0 版本文件较大，下载超时（30秒）

2. **网络超时配置**
   - 增加 `UV_HTTP_TIMEOUT`：30秒 → 300秒
   - 配置清华镜像源，提高下载速度

3. **启动配置优化**
   - 创建 `start.sh` 启动脚本
   - 配置环境变量
   - 支持 gunicorn 和 Flask 两种启动方式

## 🚀 重新部署步骤

### 第一步：提交代码修改

确保以下文件已更新：

1. **requirements.txt** - zstandard 版本已降级
2. **pyproject.toml** - 清华镜像源 + 超时配置
3. **.env** - 环境变量配置
4. **Procfile** - 启动命令
5. **start.sh** - 启动脚本

### 第二步：在微信云托管平台重新部署

1. 进入微信云托管控制台
2. 选择你的服务
3. 点击"重新部署"
4. 等待部署完成

### 第三步：验证部署成功

部署成功后，你会看到：

```
2026-03-23T20:35:47+08:00 info: [build] [runtime] Pipeline run success
2026-03-23T20:36:45+08:00 info: [launch] Deployment success
```

如果还是失败，查看具体的错误信息。

## 🔍 常见错误及解决方案

### 错误1：网络超时

**错误信息：**
```
Failed to download distribution due to network timeout
```

**解决方案：**
- ✅ 已增加 `UV_HTTP_TIMEOUT` 到 300 秒
- ✅ 已使用清华镜像源
- 重新部署即可

### 错误2：构建失败

**错误信息：**
```
Build failed: package not found
```

**解决方案：**
- 检查 `requirements.txt` 是否有语法错误
- 确保所有依赖版本正确
- 移除不必要的依赖

### 错误3：启动失败

**错误信息：**
```
Application failed to start
```

**解决方案：**
- 检查 `Procfile` 格式是否正确
- 检查 `start.sh` 是否有执行权限
- 查看应用日志获取详细错误信息

## 📊 优化建议

### 减少依赖数量

当前依赖较多（134个），可以考虑：

1. **移除不必要的依赖**
   - 检查每个依赖是否真的需要
   - 例如：如果不需要数据库，可以移除 `psycopg2-binary`

2. **合并相似依赖**
   - 例如：`psycopg` 和 `psycopg2-binary` 只保留一个

3. **使用最小化依赖**
   - 例如：`numpy` 可以使用更小的版本

### 使用更快的镜像源

当前配置：
- ✅ 清华镜像：`https://pypi.tuna.tsinghua.edu.cn/simple/`
- ✅ 阿里云镜像：`https://mirrors.aliyun.com/pypi/simple/`

清华镜像通常更快更稳定。
