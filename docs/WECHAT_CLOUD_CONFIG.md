# 微信云托管配置指南

## 🚀 部署完成后需要做的事情

### 1️⃣ 获取云托管 URL

1. 登录微信云托管：https://cloud.weixin.qq.com/
2. 进入你的服务列表
3. 点击服务名称 `wechat-bot`
4. 在服务详情页找到 **"访问设置"** 或 **"服务地址"**
5. 复制公网 URL（类似：`https://service-xxx.tcb.qcloud.la`）

**注意：** URL 是你云托管服务的域名，**不包含** `/stream_run` 路径。

---

### 2️⃣ 配置微信公众号服务器

#### 方式一：使用微信公众平台网页

1. 登录微信公众平台：https://mp.weixin.qq.com/
2. 进入 **"开发"** → **"基本配置"**
3. 找到 **"服务器配置"** 区域
4. 点击 **"修改配置"**

#### 方式二：使用微信云托管一键配置

如果微信云托管提供了"一键配置公众号"功能，直接点击使用即可。

---

### 3️⃣ 填写服务器配置

| 配置项 | 填写内容 | 说明 |
|--------|----------|------|
| **URL** | `你的云托管URL/stream_run` | 例如：`https://service-xxx.tcb.qcloud.la/stream_run` |
| **Token** | `xingtuai2026` | 固定值，代码中已定义 |
| **EncodingAESKey** | 点击"随机生成" | 微信会自动生成 43 位密钥 |
| **消息加解密方式** | **明文模式** | ⚠️ 必须选择明文模式 |
| **数据格式** | **XML** | ⚠️ 必须选择 XML |

#### URL 示例

假设你的云托管 URL 是：`https://service-abcd1234.tcb.qcloud.la`

那么正确的 URL 应该是：`https://service-abcd1234.tcb.qcloud.la/stream_run`

**注意：**
- ✅ 必须包含 `/stream_run` 路径
- ✅ 使用 `https://` 协议
- ❌ 不要添加其他路径（如 `/api` 等）

---

### 4️⃣ 提交配置

1. 填写完所有配置后，点击 **"提交"**
2. 微信会发送验证请求到你的服务器
3. 如果配置正确，会显示 **"提交成功"**

---

### 5️⃣ 启用服务器配置

提交成功后，需要点击 **"启用"** 按钮才能正式生效。

---

## ✅ 验证配置是否成功

### 方法一：查看云托管日志

1. 进入微信云托管服务页面
2. 点击 **"日志"** 或 **"运行日志"**
3. 查看是否有类似日志：

```
[2026-03-23 23:30:00] 收到微信验证请求: echostr=1234567890
[2026-03-23 23:30:00] 验证成功，返回 echostr
```

### 方法二：测试微信公众号

1. 关注你的微信公众号
2. 发送一条测试消息（如：`你好`）
3. 等待 AI 回复

## 🔧 配置环境变量（如果还没配置）

### 在微信云托管中配置

1. 进入微信云托管服务页面
2. 点击 **"环境变量"** 或 **"配置"**
3. 添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `COZE_WORKSPACE_PATH` | `/workspace/projects` | 工作目录 |
| `COZE_WORKLOAD_IDENTITY_API_KEY` | `你的Coze API密钥` | ⚠️ 必填 |
| `COZE_INTEGRATION_MODEL_BASE_URL` | `你的Coze Base URL` | ⚠️ 必填 |
| `UV_HTTP_TIMEOUT` | `300` | 超时时间 |

### 如何获取 Coze API 密钥？

1. 登录 Coze 平台：https://www.coze.cn/
2. 进入你的项目
3. 点击 **"设置"** → **"API Key"**
4. 复制 API Key 和 Base URL

## ❌ 常见问题

### Q1: 提交配置失败，提示"URL 无法访问"

**可能原因：**
- URL 错误（没有 `/stream_run` 路径）
- 云托管服务未启动
- 网络问题

**解决方案：**
1. 检查 URL 是否包含 `/stream_run`
2. 检查云托管服务状态
3. 直接在浏览器访问 URL 测试是否可访问

### Q2: 提交成功，但发送消息无回复

**可能原因：**
- 环境变量未配置（API Key 缺失）
- Coze API 服务异常
- 云托管日志有错误

**解决方案：**
1. 检查环境变量是否正确配置
2. 查看云托管日志
3. 检查 Coze API 是否正常

### Q3: 消息加密方式必须用明文模式吗？

**是的！** 当前代码只支持明文模式，暂不支持加密模式。

如果必须使用加密模式，需要修改代码支持消息加解密。
