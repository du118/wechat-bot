# 微信公众号无响应问题排查指南

## 问题描述
用户反映微信公众号部署到微信云托管后，发送消息没有反应。

---

## 🔍 问题诊断步骤

### 步骤 1：检查微信云托管服务状态

1. 登录微信云托管：https://cloud.weixin.qq.com/
2. 进入服务 `flask-enou` 或 `wechat-bot`
3. 查看 **"服务状态"** 是否为 **"运行中"**
4. 如果不是运行中，点击 **"启动"** 按钮

---

### 步骤 2：检查微信云托管日志

1. 在服务页面，点击 **"日志"** 或 **"运行日志"**
2. 查看最近的日志，重点查看：
   - 是否有启动日志
   - 是否有错误信息
   - 是否有请求日志

**正常启动日志应该包含：**
```
星途纪元AI - 微信公众号服务器启动中...
Token配置：xingtuai2026
监听地址：0.0.0.0:80
```

**如果有请求，应该看到：**
```
收到请求：GET/POST
验证请求参数：...
```

---

### 步骤 3：检查环境变量配置

⚠️ **这是最常见的问题！**

1. 进入微信云托管服务页面
2. 点击 **"环境变量"** 或 **"配置"**
3. 检查以下环境变量是否配置：

| 环境变量 | 是否必需 | 说明 |
|----------|----------|------|
| `COZE_WORKSPACE_PATH` | ✅ 必需 | 值应为 `/workspace/projects` |
| `COZE_WORKLOAD_IDENTITY_API_KEY` | ✅ 必需 | Coze API 密钥 |
| `COZE_INTEGRATION_MODEL_BASE_URL` | ✅ 必需 | Coze Base URL |
| `UV_HTTP_TIMEOUT` | ⚠️ 推荐 | 值应为 `300` |

**如果环境变量未配置，服务无法正常工作！**

---

### 步骤 4：获取 Coze API 密钥

如果环境变量未配置，你需要先获取 Coze API 密钥：

1. 登录 Coze 平台：https://www.coze.cn/
2. 进入你的项目
3. 点击 **"设置"** → **"API Key"**
4. 复制以下信息：
   - API Key
   - Base URL

**配置到微信云托管环境变量中：**
```
COZE_WORKSPACE_PATH=/workspace/projects
COZE_WORKLOAD_IDENTITY_API_KEY=你的API密钥
COZE_INTEGRATION_MODEL_BASE_URL=你的BaseURL
UV_HTTP_TIMEOUT=300
```

---

### 步骤 5：验证服务是否可访问

在浏览器中访问你的云托管 URL：
```
https://flask-enou-237504-9-1414979075.sh.run.tcloudbase.com/stream_run?signature=test&timestamp=1234567890&nonce=abcdef&echostr=test
```

**预期结果：**
- ✅ 如果服务正常运行，应该返回 `test`
- ❌ 如果返回 503 或其他错误，说明服务未正常启动

---

### 步骤 6：重新部署服务

如果环境变量有变更，需要重新部署：

1. 点击 **"部署"** → **"新建版本"**
2. 或直接点击 **"重新部署"** 按钮
3. 等待 3-5 分钟

---

### 步骤 7：检查微信公众号配置

1. 登录微信公众平台：https://mp.weixin.qq.com/
2. 进入 **"开发"** → **"基本配置"**
3. 查看 **"服务器配置"** 区域

**确认以下配置正确：**

| 配置项 | 正确值 |
|--------|--------|
| **URL** | `https://flask-enou-237504-9-1414979075.sh.run.tcloudbase.com/stream_run` |
| **Token** | `xingtuai2026` |
| **EncodingAESKey** | 随机生成的密钥 |
| **消息加密** | 明文模式 |
| **数据格式** | XML |

**⚠️ 重要提醒：**
- URL 必须包含 `/stream_run` 路径
- Token 必须是 `xingtuai2026`（不是 `xingtulai2026`）
- 消息加密必须是 **明文模式**

---

### 步骤 8：重新提交配置

如果配置有误：

1. 点击 **"修改配置"**
2. 修正 URL 和 Token
3. 点击 **"提交"**
4. 等待验证成功
5. 点击 **"启用"**

---

## 🧪 测试步骤

### 测试 1：发送简单消息

1. 关注你的微信公众号
2. 发送消息：`你好`
3. 等待 AI 回复（通常 3-10 秒）

### 测试 2：查看云托管日志

1. 进入微信云托管服务页面
2. 点击 **"日志"**
3. 查看是否有新的请求日志

**正常日志应该显示：**
```
收到请求：POST
收到POST消息：<xml>...
消息类型：text
用户消息：你好
开始调用AI助手...
AI助手回复：...
```

## ❌ 常见问题与解决方案

### 问题 1：服务返回 503

**原因：** 服务未正常启动或崩溃

**解决方案：**
1. 查看云托管日志
2. 检查环境变量是否配置
3. 重新部署服务

### 问题 2：服务启动但无法响应

**原因：** 环境变量未配置（缺少 API Key）

**解决方案：**
1. 配置 `COZE_WORKLOAD_IDENTITY_API_KEY` 和 `COZE_INTEGRATION_MODEL_BASE_URL`
2. 重新部署服务

### 问题 3：验证失败

**原因：** URL 或 Token 配置错误

**解决方案：**
1. 检查 URL 是否包含 `/stream_run`
2. 检查 Token 是否为 `xingtuai2026`
3. 重新提交配置

### 问题 4：AI 无法回复

**原因：** Coze API 密钥错误或服务异常

**解决方案：**
1. 检查 API 密钥是否正确
2. 查看云托管日志中的错误信息
3. 重新配置环境变量并部署
